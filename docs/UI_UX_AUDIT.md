# UI/UX 审查报告与改造方案

> 审查日期：2026-07-05。方式：本地运行站点，用无头浏览器对首页、题目列表、题目详情
>（含比较/解法/动态图像标签页）、思路库、比赛页做了桌面（1440×900）与移动（390×844）、
> 深色与浅色主题的全量截图走查，并结合组件源码核对交互实现。
> 截图存于工作树 `audit/` 目录。评价标准：符合"解法竞技场"定位，现代、优雅、精致。

## 总体评价

**产品骨架很强，皮肤和神经末梢欠打磨。**内容架构（解法卡的"为什么会想到/完整解析/
人工复核/已验证·未验证"、比较工具、方法边界、动态图像实验台）在同类产品里是
显著超出平均水平的设计，值得保留和强化。差距集中在三层：

1. **视觉基础层**：字体栈还是 Arial/Arial Black，品牌色在青色与靛紫间摇摆——这是
   "不够现代精致"的最大来源；
2. **完成度层**：标题里 `$P$` 不渲染、无 favicon、详情页无独立标题、空格子等破绽；
3. **反馈层**：没有加载态、没有键盘焦点态、标签栏不吸顶——这些直接放大了"慢"的体感。

深色主题的气质与"竞技场"定位契合度高；浅色主题是靠全局 override 反推出来的，
明显弱于深色。以下按层展开，每条附证据与建议。

## A. 视觉基础（现代·优雅·精致的地基）

### A1. 字体系统是精致感的最大短板【最高优先级】

`app/globals.css` 中 `body` 是 `Arial, "PingFang SC", "Microsoft YaHei"`，
`.font-display` 是 `Arial Black, Arial`。Arial/Arial Black 是上世纪的 Web 安全字体，
撑不起现代感；大标题（ProofArena、题目擂台）用 Arial Black 渲染，笔画粗糙、
字距生硬，与 KaTeX 优雅的衬线数学字体并置时反差刺眼。

**建议**：
- 拉丁/数字：用 `next/font` 自托管一款几何感现代无衬线（Inter、Geist、
  Space Grotesk、IBM Plex Sans 任选其一；display 与正文可同族不同字重）。
- 中文：明确声明现代中文字体栈
  `"PingFang SC", "MiSans", "HarmonyOS Sans SC", "Noto Sans SC", sans-serif`
  （中文继续走系统字体，不必下载 web font，成本为零但顺序要写对）。
- 所有统计数字、评分（9.6、19、44）加 `font-variant-numeric: tabular-nums`，
  数字列对齐是"精密仪器感"的关键细节。
- 用 `next/font` 而非 CSS import，避免字体闪换（FOUT）。

### A2. 品牌色在两个主题间摇摆

深色主题主色是 cyan-400（logo 块、CTA、tab 高亮），浅色主题下同一批元素被
override 成靛紫/indigo（`::selection` 是 `#4f46e5`，按钮呈紫色），同页还混有
红（压轴）、amber（比赛）。品牌无法沉淀出"一眼认出"的颜色。

**建议**：把颜色收敛为语义 token（CSS 变量随 `data-theme` 切换）：
`--accent`（品牌主色，两主题同色相、不同明度）、`--danger`（压轴/风险）、
`--contest`（amber，比赛专属）、`--verified`（绿，已验证）。浅色主题不再靠
全局类名 override 反推，而是直接消费同一组 token。直角零圆角的锐利风格
符合竞技定位，建议保留，但浅色下 `border-white/10` 映射出来的灰边偏脏，
需要单独调一档浅色边框/表面色阶。

### A3. 首页 2.5MB 背景大图视觉贡献接近零

深色下它几乎不可见（只剩右侧一团微光），浅色下被渐变冲淡成噪点。
花了 2.5MB 带宽（见性能报告 P4）没换来任何构图价值。

**建议**：直接删图，用代码造氛围——已有的 `grid-surface` 网格 + 一两个
`radial-gradient` 光晕 + 细微噪点纹理即可，成本 0KB 且两主题各自可调。
这一条与性能优化 P4 一举两得。

## C. 交互与反馈（"现代"的核心是即时反馈）

### C1. 没有任何加载反馈【与性能问题直接联动】

全站没有一个 `loading.tsx`。当前动态渲染下页面切换要 0.5–2s，点击链接后
界面毫无反应——这是用户"感觉网站慢"的直接放大器。即使性能报告的静态化
落地后，仍建议保留骨架屏兜底。

**建议**：为 `/problems`、`/problems/[id]`、`/contests` 添加 `loading.tsx`
骨架屏（灰块模拟题目卡结构，250ms 淡入避免闪烁）；导航链接用
`useLinkStatus`（Next 15.3+）或顶部细进度条给出即时 pending 反馈。

### C2. 键盘焦点态缺失

主站交互组件（头部导航、标签页、卡片、展开按钮）没有 `focus-visible` 样式，
键盘用户完全迷路，这也是精致度的硬指标。

**建议**：全局
`:focus-visible { outline: 2px solid var(--accent); outline-offset: 2px; }`，
配合 A2 的 token 两主题自动适配。

### C3. 无 `prefers-reduced-motion` 处理

`html { scroll-behavior: smooth }` 全局生效且无法退出。

**建议**：包一层
`@media (prefers-reduced-motion: no-preference) { html { scroll-behavior: smooth } }`；
后续所有动效统一遵守。

### C4. 详情页标签栏不吸顶

题目/比较/解法/知识点/相关题/动态图像 六个标签是详情页的骨干导航，
但滚动后随页面滚走，长内容（展开的完整解析）里用户失去位置感。

**建议**：标签栏 `sticky top-16`（紧贴站头下沿），滚动时保持可见；
当前激活标签加下划线滑动指示。

### C5. 移动端标签横向溢出无提示

390px 宽下"动态图像"被裁切在视口外，无任何"右侧还有内容"的暗示。

**建议**：溢出容器右缘加渐变遮罩 + 允许惯性横滚；或小屏改两行换行。

### C6. 悬浮比赛卡永久遮挡内容【移动端实锤】

`app/page.tsx:197` 的比赛推广卡 `fixed bottom-4 right-4 z-50` 无关闭按钮，
移动端宽度近全屏（`w-[calc(100vw-2rem)]`），实测盖住了"默认阅读路径"标题。

**建议**：加关闭按钮 + `localStorage` 记忆（关闭后本次赛季不再弹）；
或收敛成右下角小胶囊（奖杯图标 + "比赛进行中"），点击展开。

### C7. 原生 `<select>` 与整体风格脱节

题目列表的筛选（全部卷别/全部专题）和解法 Diff 的路线选择用的是浏览器
原生下拉，圆角、白底、系统字体，与全站直角锐利风格割裂。

**建议**：换自定义 listbox（Radix Select / Headless UI，或手写
`role="listbox"`），沿用直角+边框语言；同时解决移动端原生下拉样式不可控问题。

### C8. 微交互缺一套统一规范

卡片 hover 只有边框变色，按钮没有 pressed 态，各处 `transition` 时长随意。

**建议**：定一组 motion token（`--transition-fast: 150ms cubic-bezier(0.2,0,0,1)`），
按钮加 `active:translate-y-px`，卡片 hover 统一"边框亮 + 极轻阴影"，克制不炫技。

## D. 信息架构与布局

### D1. 题目列表页每张卡都是"迷你详情页"【核心改造点】

实测 19 道题的列表页整页高 **8135px**，每卡约 400px：徽章、题干（含公式）、
解法速览双栏、学习指数、标签、投票数全部平铺。列表页的职责是**扫描与选择**，
不是阅读——现在用户找一道题要滚十几屏。

**建议**：默认紧凑行——`题号 | 标题 | 卷别·年份徽章 | 难度 | 解法数 | 学习指数`
一行放下（桌面约 64px 高），hover 或点击行内展开"解法速览"；
提供"紧凑/详细"视图切换记忆到 localStorage。右缘的题号进度轨是好设计，保留。

### D2. 详情页题面重复渲染

"题目本体"卡片（页面上部）与标签区左栏的"对照题目"卡在同一屏出现两次
完整题干，且对照卡下方整列空白。

**建议**：左栏改为"滚动跟随的锚点目录 + 收起式题面回看"（点击才展开题干），
既省重复又填补空白列。

### D3. "比较"标签内三层嵌套导航

主标签（6 个）→ 比较工具子标签（8 个：发布信息/解法比较/方法边界/推导过程/
解法挑战/解法 Diff/图谱来源/思路树）→ 各自内部折叠面板。学生用户很难建立地图。

**建议**：8 个子工具归并为 3 组纵向区块——**总览**（解法比较表+方法边界）、
**深入**（推导过程+挑战+Diff）、**溯源**（图谱来源+思路树+发布信息），
改为带 scroll-spy 的左侧锚点导航，一条滚动流读完，比嵌套标签更符合阅读心智。

### D4. 桌面首页 hero 右半近乎全空

标题左对齐后右侧约半屏纯背景（背景图又几乎不可见，见 A3）。

**建议**：右侧放一个"动态战况"模块：本周对比最多的题、最新挑战、一条动画
演示的公式变换——既填空间又点题"竞技场"。

### D5. 比赛页只有一张卡，下方大量空白

**建议**：补"比赛怎么玩"（三步说明）、"评审规则"摘要、往期回顾占位——
让单场比赛时期页面也有完整叙事。

### D6. 页脚过于简陋

一行 demo 声明 + 关于/投稿两个链接，与内容的用心程度不匹配。

**建议**：标准三栏页脚：栏目导航 / 开源与许可（GitHub、CC 协议——仓库有
LICENSE-CONTENT）/ 联系与投稿入口；若将来国内部署补 ICP 备案位。

## 值得保留的亮点（改造时不要破坏）

- 解法卡的内容分层：为什么会想到 → 完整解析摘要 → 人工复核（已验证/未验证）——
  这是产品的灵魂，信息设计成熟。
- 动态图像实验台（怎么操作/看懂什么引导框）。
- 深色主题的整体氛围、直角锐利的视觉语言。
- 列表页右缘题号进度轨、移动端表格降级、题目详情的关键条件侧栏。
