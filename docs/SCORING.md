# 五维评分规则

评分的目的不是制造一个绝对排名，而是公开解释一条解法的优势与代价。

每项使用 0-10 分。正式题解内容通常使用一位小数。

## 1. 正确性 `correctness`

衡量推理链是否完整可靠。

| 分数 | 参考标准 |
| --- | --- |
| 9.5-10 | 结论正确，条件、边界、分类和取等无遗漏，已独立复核 |
| 8.5-9.4 | 主线正确，存在少量可补足但不改变结论的省略 |
| 7.0-8.4 | 思路可行，但关键步骤缺少说明或验证 |
| < 7.0 | 不应进入正式内容，应先修正 |

正确性不是风格分。尚未验证的漂亮思路不能获得接近满分。

## 2. 考场性 `examReady`

衡量高中生在限定时间内识别、书写和检查该路线的可行性。

考虑：

- 起手是否容易识别
- 是否符合常见评分点
- 计算是否稳定
- 是否依赖偶然灵感
- 是否容易在压力下复现

## 3. 结构美感 `elegance`

衡量转化是否自然、结构是否简洁、是否抓住题目本质。

结构美感不等于步骤最少。隐藏关键论证或使用不必要的超纲工具不会自动提高结构美感。

## 4. 计算量 `calculation`

这里的高分表示计算负担较小、出错面较少。

考虑：

- 展开与化简长度
- 重复运算
- 分类数量
- 数值复杂度
- 能否利用结构避免硬算

## 5. 讲解友好 `explanation`

衡量解法是否适合教学、复盘和迁移。

考虑：

- 是否解释“为什么想到”
- 关键转化是否清楚
- 步骤间因果是否明确
- 能否提炼可迁移模式
- 是否方便板书或口头复述

## 参考均分

当前参考均分是五项的算术平均：

```text
(正确性 + 考场性 + 结构美感 + 计算量 + 讲解友好) / 5
```

它只用于辅助比较。用户应优先阅读解法画像、适用场景、五维分布和评分理由。

## 评分理由

每条解法必须有一句或一段评分理由，至少说明：

- 最大优势
- 主要代价
- 为什么与另一条解法产生分差

好例子：

> 紧扣目标量，避免完整求出两个交点，结构感最强；但提出顶点对应的根并消元，需要较熟练的解析几何意识。

不合格例子：

> 这是一种很好的解法，所以评分较高。

## 超纲工具

使用超纲工具不影响正确性，但通常会影响考场性，并需要明确标注。

例如：

- 泰勒展开
- 洛必达法则
- 复分析
- 竞赛专用定理
- CAS 黑盒求解

课后理解路线可以使用这些工具，但不能伪装成高考标准答法。

## 校准流程

新增解法时建议：

1. 先独立评分，不参考页面顺序
2. 与同题其他解法逐项比较
3. 检查分差是否能被评分理由解释
4. 由第二位审核者复核正确性和考场性
5. 如评分规则发生变化，更新本文档
